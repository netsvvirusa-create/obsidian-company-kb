### Nested variable resolution

Variables 35-37 have **nested identifiers**: the ID itself contains
`{{VAR|Номер варианта выполняемых работ|N}}`.

Resolution is recursive (inside-out):
1. First resolve inner `{{VAR|Номер варианта выполняемых работ|1}}` → `1`
2. Then resolve outer `{{VAR|Вариант выполняемых работ 1|...}}`

## Grammatical case requirements

| Variable | Case | Context |
|----------|------|---------|
| Должность уполн. * | Genitive (Р.п.) | «в лице Генерального директора» |
| ФИО уполн. * | Genitive (Р.п.) | «Ивановой Марии Петровны» |
| Основание * | Genitive (Р.п.) | «на основании Устава» |
| Полная ОПФ * | Nominative (И.п.) | «Общество с ограниченной ответственностью» |
| Вид стороны * | Nominative (И.п.) | «именуемое ... «Исполнитель»» |

Values in YAML are stored in the REQUIRED case (fields ending with `_рп`
for genitive case — "родительный падеж").
