# Relationships

Relationships between people are ALWAYS bidirectional. When a relationship
A→B is created, the reverse B→A must also be created.

## Relationship types and mirror pairs

| является супругом/ой | является супругом/ой |

| является свёкром/свекровью | является невесткой |
| является братом/сестрой | является братом/сестрой |

## Tag mapping

| Category | Tag | Descriptions |
|----------|-----|-------------|
| Руководство | `#связь/руководство` | руководит, подчиняется, наставник, куратор |
| Семья | `#связь/семья` | супруг/а, родитель, ребёнок, брат/сестра, зять, тесть |
| Деловая | `#связь/деловая` | совладелец, партнёр, инвестор, бывший коллега, рекомендатель, основной контакт |
| Дружба | `#связь/дружба` | друг, близкий знакомый, сосед |
| Учёба | `#связь/учёба` | однокурсник, научный руководитель, ученик |
| Прочее | `#связь/прочее` | любое другое описание |

## Relationship storage format

In YAML frontmatter `связи` property:
```yaml
связи:
  - "[[Петров Пётр]]"
  - "[[Сидорова Мария (ООО Пример)]]"
```

In body section `## Связи и отношения`:
```markdown
| Лицо | Тип связи | Описание | Период |
|------|-----------|----------|--------|
| [[Петров Пётр]] | Руководство | Является руководителем | 2026— |
```

## Cross-boundary relationships

Relationships work across employee ↔ contact boundary. An employee
(04-СОТРУДНИКИ) can be linked to a contact (05-КОНТАКТЫ) and vice versa.

## Automation

Use `scripts/relationship_sync.py` to check and fix bidirectional consistency:
```bash
python scripts/relationship_sync.py --vault PATH --dry-run
python scripts/relationship_sync.py --vault PATH --fix
```
