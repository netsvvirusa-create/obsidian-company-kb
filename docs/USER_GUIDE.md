## 1. Подготовка окружения

### Системные требования

- Python 3.10 или выше
- Obsidian (последняя версия)
- AI-агент: Claude Code, Codex CLI или OpenCode

### Установка зависимостей

```bash
pip install pyyaml python-docx pymorphy3 pymorphy3-dicts-ru vobject
```

Для разработки и тестирования:

```bash
pip install -r requirements.txt
```

## 2. Инициализация хранилища

Создайте новое хранилище Obsidian или выберите существующее. Запустите скрипт инициализации:

```bash
bash scripts/init_vault.sh /путь/к/хранилищу
```

Скрипт создаст:
- 15 папок для разных типов данных (контрагенты, договоры, проекты и т.д.)
- 9 дашбордов `.base` в папке `11-БАЗЫ/`
- 4 визуальные карты `.canvas` в папке `12-КАНВАСЫ/`
- Индексные заметки `_MOC.md` в каждой основной папке

Откройте хранилище в Obsidian и убедитесь, что все папки на месте.

## 3. Настройка карточки компании

Создайте заметку с реквизитами вашей компании. Это обязательный шаг для генерации .docx документов.

Файл: в корне хранилища или в `01-КОНТРАГЕНТЫ/`

```yaml
---
title: "ООО «Ваша компания»"
type: наша_компания
опф_полная: "Общество с ограниченной ответственностью"
опф_краткая: "ООО"
название_полное: "«Ваша компания»"
название_краткое: "«Ваша компания»"
адрес: "Юридический адрес"
инн: "0000000000"
кпп: "000000000"
расчётный_счёт: "00000000000000000000"
банк: "АО «Банк»"
бик: "000000000"
корр_счёт: "00000000000000000000"
tags:
  - тип/компания
---
```

Заполните все поля реальными реквизитами. Они будут автоматически подставляться при генерации договоров.

## 4. Создание контрагентов

Контрагенты хранятся в папке `01-КОНТРАГЕНТЫ/`.

Имя файла: `Название организации.md`

Обязательные поля:
- `type: контрагент`
- `инн` -- для проверки дубликатов
- `категория` -- клиент, поставщик, партнёр, подрядчик, государственный
- `статус` -- активный, приостановлен и т.д.

Заполните реквизиты: ИНН, ОГРН, адреса, банковские данные. Эти данные используются при генерации .docx документов.

В разделе "Контактные лица" добавляйте ссылки на карточки контактов через `[[wikilinks]]`.

---

## 5. Работа с контактами

Контактные лица контрагентов хранятся в `05-КОНТАКТЫ/`.

Имя файла: `Фамилия Имя (Контрагент).md`

Обязательные поля:
- `type: контакт`
- `контрагент` -- ссылка `[[wikilink]]` на контрагента
- `роль` -- должность контактного лица
- `статус`

При создании нового контакта:
1. Создайте карточку контакта в `05-КОНТАКТЫ/`
2. Добавьте ссылку `[[Фамилия Имя (Контрагент)]]` в карточку контрагента (поле `контактные_лица` и таблицу)
3. При необходимости добавьте в карточку договора

Конфиденциальные данные (личный телефон, домашний адрес) оборачивайте в сворачиваемые блоки:

```markdown
> [!warning]- Конфиденциальная информация
> Личный телефон: +7 ...
```

## 6. Работа с сотрудниками

Сотрудники вашей компании хранятся в `04-СОТРУДНИКИ/`.

Имя файла: `Фамилия Имя.md`

Обязательные поля:
- `type: сотрудник`
- `должность`
- `статус`

Дополнительные поля: `отдел`, `дата_найма`, `руководитель`, `связи`.

Связи между сотрудниками (руководство, наставничество и т.д.) всегда двусторонние -- см. раздел "Связи между людьми".

## 7. Создание договоров

Договоры хранятся в `02-ДОГОВОРЫ/`.

Имя файла: `Договор №XXX.md`

Обязательные поля:
- `type: договор`
- `номер`
- `контрагент` -- ссылка на карточку контрагента
- `дата_подписания`
- `статус`

### Поля для генерации .docx

Для генерации договоров заполните дополнительные поля:

- `вид_стороны_1`, `вид_стороны_2` -- например, "Исполнитель" и "Заказчик"
- `подписант_наш` -- ссылка на подписанта
- `должность_подписанта_нашего_рп` -- должность в **родительном падеже**
- `фио_подписанта_нашего_рп` -- ФИО в **родительном падеже**
- `основание_подписанта_нашего_рп` -- основание в **родительном падеже**
- Аналогичные поля для подписанта контрагента

Суффикс `_рп` означает "родительный падеж". Пример:
- Именительный: "Генеральный директор Иванов Иван Иванович"
- Родительный (_рп): "Генерального директора Иванова Ивана Ивановича"

### Спецификации

Спецификации к договору описываются в поле `спецификации` как список:

```yaml
спецификации:
  - номер: 1
    дата: "2026-04-10"
    тип: услуги
    услуги:
      - название: "Годовая поддержка"
        отчётность: "Акт о выполненных работах"
        стоимость: 150000
    место: "г. Москва"
    порядок_оплаты: "100% предоплата"
    сроки: "1 месяц с момента оплаты"
    гарантия: "1 год"
```

## 8. Генерация документов .docx

### Генерация договора

```bash
python scripts/generate_contract.py --vault /путь/к/хранилищу --contract "Договор №001"
```

Скрипт:
1. Читает заметку договора и разрешает ссылки на контрагента и компанию
2. Собирает 32 переменные для подстановки
3. Заполняет .docx шаблон
4. Сохраняет в `14-ВЛОЖЕНИЯ/Документы/`

Для предварительного просмотра подстановок:

```bash
python scripts/generate_contract.py --vault /путь --contract "Договор №001" --dry-run
```

### Генерация спецификации

```bash
python scripts/generate_specification.py --vault /путь --contract "Договор №001" --spec 1
```

### Проверка грамматики

Добавьте флаг `--check-grammar` для автоматической проверки падежей:

```bash
python scripts/generate_contract.py --vault /путь --contract "Договор №001" --check-grammar
```

Или проверьте существующий документ:

```bash
python scripts/grammar_check.py --file /путь/к/документ.docx --verbose
```

## 9. Проекты и переговоры

### Проекты

Папка: `03-ПРОЕКТЫ/`. Имя файла: `Название проекта.md`

Связывайте проект с клиентом, договором, руководителем и командой через `[[wikilinks]]`.

### Переговоры

Папка: `06-ПЕРЕГОВОРЫ/`. Имя файла: `ГГГГ-ММ-ДД Краткое название.md`

Фиксируйте участников, повестку, решения и следующие шаги. Используйте блок-идентификаторы `^id` для ссылок на конкретные договорённости.

## 10. Стратегия: цели и идеи

### Цели

Папка: `09-СТРАТЕГИЯ/Цели/`. Имя файла: `Цель - Название.md`

Поля: `горизонт` (квартал, год, 3 года), `направление`, `дедлайн`, `метрика_успеха`.

### Идеи

Папка: `09-СТРАТЕГИЯ/Идеи/`. Имя файла: `Идея - Название.md`

Поля: `потенциал` (высокий, средний, низкий), `направление`.

## 12. Использование дашбордов

Дашборды `.base` находятся в `11-БАЗЫ/`. Откройте любой файл в Obsidian для просмотра данных в табличном виде.

| Дашборд | Содержимое |
|---------|-----------|
| Активные договоры | Договоры с формулами "дней до окончания" и "просрочен" |

| Операционные события | Группировка по категории, фильтр критических |

## 14. Импорт данных

### Импорт из CSV

```bash
python scripts/import_csv.py --vault /путь --type контрагент --file данные.csv
```

Поддерживаемые типы: `контрагент`, `контакт`, `сотрудник`.

Скрипт автоматически распознаёт колонки по заголовкам (поддерживаются русские и английские названия). Дубликаты по ИНН (для контрагентов) и по имени файла пропускаются.

Для предварительного просмотра:

```bash
python scripts/import_csv.py --vault /путь --type контрагент --file данные.csv --dry-run
```

### Импорт из vCard

```bash
python scripts/import_vcard.py --vault /путь --file контакты.vcf --counterparty "ООО Пример"
```

Контакты создаются в `05-КОНТАКТЫ/`, ссылки автоматически добавляются в карточку указанного контрагента.

## 16. Валидация и обслуживание

### Проверка хранилища

```bash
python scripts/validate_vault.py --vault /путь
```

Проверяет:
- Наличие обязательных полей в frontmatter
- Соответствие тегов таксономии
- Битые wikilinks
- Дубликаты ИНН и номеров договоров
- Просроченные активные договоры
- Согласованность двусторонних связей

### Аудит ссылок

```bash
python scripts/audit_links.py --vault /путь         # только отчёт
python scripts/audit_links.py --vault /путь --fix    # создать заглушки
```

### Массовое обновление статусов

```bash
python scripts/bulk_status_update.py --vault /путь --folder 02-ДОГОВОРЫ --status завершён
```

При установке статуса "завершён" или "отменён" заметки автоматически перемещаются в `99-АРХИВ/`.

## 17. Связи между людьми

Связи между сотрудниками и контактами **всегда двусторонние**. Если А руководит Б, то у Б должна быть обратная запись "подчиняется А".

### Добавление связи

1. Добавьте `[[ссылку]]` в поле `связи` в YAML обеих заметок
2. Добавьте строку в таблицу `## Связи и отношения` обеих заметок с зеркальным описанием

### Зеркальные описания

| Если A -> B | То B -> A |
|-------------|-----------|
| руководит | подчиняется |
| является наставником | является подопечным |
| является родителем | является ребёнком |
| является другом | является другом |
| рекомендовал | рекомендован |

Полный список -- в файле `references/RELATIONSHIPS.md`.

### Автоматическая синхронизация

```bash
python scripts/relationship_sync.py --vault /путь --dry-run   # предпросмотр
python scripts/relationship_sync.py --vault /путь --fix        # исправить
```
